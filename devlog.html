DarkV Studios — Devlog

The real development history of our projects: technical challenges, unique features, failures, and solutions discovered during the coding process.

Streamer Summer — Streamer Simulator

A pixel-art life simulator where the player goes from using a cheap webcam to reaching top viral rankings within 90 days.

Challenge: The hardest task was writing a massive number of unique chat questions that would not repeat, as well as balancing the internal economy so the player could not complete the entire game in one evening.
Solution: Created a dedicated line database for chat messages and implemented flexible income coefficients based on the current mood of the viewers.
Feature: A dynamic interactive chat system. By the way, the game is currently in its final stage and is undergoing Google's review process right now!

Spend your summer — Idleness Simulator

An addictive 2D idle clicker where the main goal is to procrastinate as effectively as possible. However, behind the simple graphics hid serious problems with level balancing.

Challenge: It was incredibly difficult to connect and configure items of different levels during upgrades (for example, when all items are level 2, but the player buys a level 3 table). Integrating Google Ads was also a major pain point.
Solution: Rewrote the UI update logic and manually balanced every upgrade level. As for the advertisements, it required digging through a mountain of documentation until the Google Ads plugin finally worked stably.
Feature: The day-draining mechanic. The game has a strict time limit of 90 days. The player must "waste" this time as quickly as possible by purchasing upgrades that accelerate the passing days.

TexnoMarket sim — 3D Store Simulator

A large-scale management simulator that became a serious technical and creative challenge for the studio.

Challenge: It was very difficult to optimize the game so it could run on low-end PCs, because rendering over 50 products hit the FPS severely. Another pain point was the customer queue, which broke constantly.
Solution: To fix the queue, many custom methods were tested, but eventually, a stable third-party solution from the Asset Store was integrated, resolving the AI collision problems.
Feature: This project was more of an experiment, because I used only artificial intelligence to make this game, which was quite difficult but interesting!

Mirror Room — 3D Puzzle on Unity 6

An elegant game about finding differences in mirrored worlds, which forced a complete rethink of how cameras work in a 3D space.

Challenge: It was difficult to ensure that adjacent rooms in both worlds did not intersect or block one another. After all, the player isn't actually walking across the map — the map is moving under the player!
Solution: The issue was solved using a Render Texture system. The player does not click on the 3D objects directly, but on a special plane that projects a live view from an isolated camera.
Feature: A clean and unique "mint-jelly" UI style, fully implemented using the latest Unity 6 engine features.

Coffee Rush Tycoon — Low-poly Tycoon

My first project developed completely on my own. While many beginner mistakes were made, they provided priceless experience.

Challenge: The order system would break if the receipt contained only 1 or 2 items (it only functioned properly with exactly 3 items). Also, ingredient spawning inside the refrigerator had a mind of its own.
Solution: Adjusted the exact coordinates of the spawn points, and solved the order problem by blocking all other combinations and restricting orders strictly to 3 items.
Feature: A completely independent architecture for the coffee-making logic and a deep upgrade system for professional espresso machines.

Develop Simulator — Text Management

An indie studio management simulator where the player's main goal is to avoid bankruptcy and handle team stress levels.

Challenge: Adjusting the mathematical balance of income and expenses was an extremely complicated task. Another difficulty was adapting the interface for various screen form-factors, since Unity does not handle this automatically.
Solution: Put a lot of effort into refining the mathematical formulas to ensure the game felt fair and fun. Screens were adapted by configuring custom Canvas Anchors in Unity.
Feature: A realistic developer burnout system and random events that can pull the studio under in less than a minute.

Color Dash! — Physics Puzzle

A ball-rolling physics game where using different paint types allows the user to alter physical properties directly on the stage.

Challenge: The hardest part was making the controls and the drawing process intuitive and clear for the player, so that guiding the ball into the hole was actually achievable. The mathematics behind calculating star ratings was also tough to get right.
Solution: Rewrote screen touch detection for more accurate line drawing, and tied the star system directly to the remaining level time and the amount of paint consumed.
Feature: Unique material properties: magnetic, icy, and bouncy paints that completely transform object behavior on the fly.

Bombordilo Attack! — Meme 3D Runner

A fast-paced, chaotic 3D runner featuring popular Italian meme animals, created purely for good vibes and entertainment.

Challenge: The biggest nightmare in this project was setting up the advertisements. Because I was doing it for the first time, I initially followed some sketchy and outdated guides online that did not work at all.
Solution: Completely removed the old scripts, closed the broken tutorials, studied the official SDK implementation, and successfully integrated the banner ads.
Feature: This game was made completely for fun! Absurd humor, insane pacing, and unique sound effects from viral memes.
